# Heritage Unified

Heritage Public School-এর **হোস্টেল ম্যানেজমেন্ট** ও **HAIS (একাডেমিক ইন্টেলিজেন্স সিস্টেম)** —
এই দুটো স্বাধীন অ্যাপকে একীভূত করার প্রজেক্ট। বিদ্যমান দুটো অ্যাপের একটি ফাইলও এই রিপোতে নেই এবং
এই রিপোর কোনো কিছু ডিপ্লয় করলে সেই দুটো অ্যাপ **অপরিবর্তিত** থাকে।

## রিপো স্ট্রাকচার

```
heritage-unified/
├── roster-sync/                 ← ছাত্র-তালিকা অটো-সিঙ্ক (Cloud Functions, ব্যাকএন্ড-অনলি)
│   ├── heritage-hostel/         ← হোস্টেল Firebase প্রজেক্টে ডিপ্লয় হবে
│   └── abu-sayed-6c6b9/         ← HAIS Firebase প্রজেক্টে ডিপ্লয় হবে
└── unified-shell/               ← নতুন গেটওয়ে অ্যাপ (শিক্ষকরা এখনো এটা ব্যবহার করছেন না)
    └── public/index.html
```

## ডিপ্লয়মেন্টের ক্রম

**অবশ্যই এই ক্রমে করুন** — ধাপ ধাপে যাচাই করে এগোনোই নিরাপদ। বিস্তারিত `DEPLOYMENT.md`-এ।

1. `roster-sync/heritage-hostel` ডিপ্লয় ও টেস্ট
2. `roster-sync/abu-sayed-6c6b9` ডিপ্লয় ও টেস্ট
3. দুই দিক থেকে add/delete টেস্ট করে সিঙ্ক যাচাই
4. `unified-shell` কনফিগার ও ডিপ্লয়
5. প্রথম অ্যাডমিন রোল ম্যানুয়ালি বসানো
6. পাইলট টেস্ট (নিজে + ১-২ জন বিশ্বস্ত ব্যবহারকারী)

## নিরাপত্তা নিশ্চয়তা
- বিদ্যমান হোস্টেল অ্যাপ ও HAIS-এর কোনো ফাইল এই রিপোতে সংযোজন/পরিবর্তন করা হয়নি
- `roster-sync`-এর দুটো ফাংশন শুধু ছাত্রের **নাম-তালিকা** সিঙ্ক করে — অন্য কোনো ডেটা (হাজিরা, নম্বর,
  আচরণ) স্পর্শ করে না
- `unified-shell` একটা সম্পূর্ণ নতুন URL/ফাইল — যতক্ষণ শিক্ষকদের এই লিংক ও রোল না দেওয়া হচ্ছে,
  ততক্ষণ তারা এর অস্তিত্বই জানবেন না এবং পুরনো অ্যাপেই কাজ চালিয়ে যেতে পারবেন
